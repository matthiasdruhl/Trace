# Project Trace: Research & Strategic Planning
## The "Zero-Laptop" Pre-Build Phase

### 1. Objective
This document outlines the high-level architectural research and product planning to be completed before the coding phase begins. These tasks are designed to be performed using mobile devices and browser-based tools, focusing on validating the core technical and business assumptions for the Codex Creator Challenge.

### 2. High-Level Research Stack (Mobile-Ready)
* **DuckDB Shell (Wasm):** Test metadata filtering logic on S3-hosted Parquet/CSV files directly in a mobile browser.
* **Arxiv / Technical Papers:** Research "Partition-Aligned Vector Search" and "IVF-PQ optimization for Object Storage."
* **Excalidraw:** Create systems architecture diagrams (Compute-Storage decoupling) for the pitch deck.

### 3. Uber Michelangelo Deep-Dive
Targeting Uber's internal ML standards to align with competition sponsors:
* **TTE (Two-Tower Embeddings):** Investigate how Trace can index historical data generated by Michelangelo’s TTE models.
* **DOMA (Domain-Oriented Microservices Architecture):** Position Trace as a storage domain extension.
* **Audit & Compliance:** Research city-level regulatory data structures (NYC/SF/London) to build a realistic "Synthetic Uber Dataset."

### 4. Zero-Code Deliverables (Phase 1)
| Task | Description | Tool |
| :--- | :--- | :--- |
| **System Spec** | Detailed JSON schema for the MCP Server skill. | Gemini / Mobile Text Editor |
| **Cost Model** | Comparison of Trace Lambda costs vs. Databricks/Pinecone. | Google Sheets (Mobile) |
| **Product UI** | High-fidelity mockups of the Trace "Heatmap" dashboard. | Figma (Mobile) / Canva |
| **Synthetic Data** | 1,000+ entries of mock regulatory/audit logs for demoing. | Gemini (Prompt Generation) |

### 5. Technical Decision Matrix
* **Index Portability:** Confirming the **Lance** format's random-access performance on ARM64/Graviton.
* **Vector Math:** Documenting SIMD/NEON intrinsics for ARM to be passed to the "C-Expert" or Codex.
* **Security:** Drafting the "In-VPC" deployment architecture for enterprise data sovereignty.

### 6. Preparation for Week 1 (The Build)
* Create the `project_spec.md` with final API endpoints.
* Define the exact byte-range request logic for S3 headers.
* Identify the top 5 "Design Partners" (Law firms, Audit firms) in NYC for pilot feedback.
